# ThreatWeaver

ThreatWeaver is a correlated Zero Trust security engine that fuses ML anomaly detection, behavioral rules, MITRE ATT&CK mapping, and constrained AI reporting to convert raw AD logs into structured, explainable, and evidence-driven threat intelligence.

It’s designed as a practical, evidence-first pipeline:

1. **Ingest** AD/DC + client endpoint logs (CSV)  
2. **Detect & correlate** using **Isolation Forest** + **behavioral/rule detections**  
3. **Map** detections to **MITRE ATT&CK** techniques  
4. **Score & prioritize** incidents with a deterministic risk/confidence model  
5. **Present** results via a lightweight **SOC dashboard** (Flask + HTML templates)  
6. **Generate** a constrained, SOC-style narrative report via a local LLM (Ollama)

---

## Key capabilities

- **Hybrid detection**: ML anomaly detection (Isolation Forest) + rule/behavior detections
- **Correlation & prioritization**: deterministic threat scoring and risk tiers (LOW/MEDIUM/HIGH/CRITICAL)
- **MITRE ATT&CK mapping**: attaches technique labels to events based on observed behaviors
- **Explainable outputs**: exports enriched analysis CSV + high-risk incident CSV for SOC workflows
- **SOC dashboard**: REST endpoints + UI to explore summary stats, top incidents, MITRE distribution, timelines, etc.
- **SENTINEL AI reporting**: optional LLM-generated report, constrained by the “machine verdict” logic

## Repository structure (high level)

Core pipeline:
- `train_isolation_forest_v4.py` — main analysis pipeline (loads CSVs → detects → scores → exports)
- `advanced_security_analysis.csv` — enriched output (generated by pipeline; sample may exist in repo)
- `high_risk_incidents.csv` — filtered HIGH/CRITICAL events output (generated by pipeline; sample may exist)

Dashboard:
- `dashboard.py` — Flask backend (serves `templates/dashboard.html` + `/api/*` endpoints)
- `templates/` — HTML templates (dashboard UI)
- `static/` — JS/CSS assets for the dashboard UI

Secure log transfer (optional demo utilities):
- `secure_log_server.py` — HTTPS upload receiver (Flask)
- `secure_log_uploader.py` — periodic client uploader
- `generate_cert.py` — certificate helper (if used in your environment)

Simulation helpers:
- `attack.sh` / `auto_attack_simulator.ps1` — attack simulation scripts (intended for lab/demo use)

## Quick start (local)

### 1) Prerequisites

- Python 3.9+ recommended
- Suggested Python packages (based on code imports):
  - `pandas`
  - `numpy`
  - `scikit-learn`
  - `flask`
  - `requests`

> Tip: If you don’t have a `requirements.txt` yet, you can install manually:
>
> ```bash
> pip install pandas numpy scikit-learn flask requests
> ```

### 2) Prepare input log data

The pipeline expects these files in the repository root:

- `dc_logs.csv`
- `client_logs.csv`

Minimum required columns (validated by the pipeline):
- `timestamp`
- `event_id`
- `username`

Optional columns (the pipeline will create defaults if missing):
- `process_name`
- `command_line`
- `source_ip`

```bash
python train_isolation_forest_v4.py
